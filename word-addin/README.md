# Refx Word Add-in

This folder contains a Microsoft Word task pane add-in for inserting Refx references into Word documents.

The important design choice is that visible citation numbers are never the source of truth. The add-in stores stable citation state in a Word custom XML part, places each visible citation inside a Word content control, and recalculates numeric labels from document order whenever you refresh.

## Hosting Modes

- Development frontend: `https://localhost:5174`
- Production homepage: `https://refx.667764.xyz/`
- Production add-in frontend: `https://refx.667764.xyz/word/`
- Local Refx bridge, both modes: `http://127.0.0.1:38474`

Use `manifest.xml` for local development. Use `manifest.production.xml` for the hosted production add-in.

## Supported Word Host

This add-in is intentionally supported in desktop Microsoft Word only: Word for Windows and Word for macOS.

Word for the web is blocked by the task pane. During testing, Word web rejected or destabilized the content-control operations that Refx needs for reliable citation state, autosave, renumbering, and bibliography rebuilds. Refx therefore does not use a plain-text/browser fallback for real documents.

## Project Structure

- `src/api/refxClient.ts` calls the local Refx bridge at `http://127.0.0.1:38474`.
- `src/word/customXmlStore.ts` reads/writes the document-level custom XML citation state.
- `src/word/citationControls.ts` creates and updates `refx-citation` and `refx-bibliography` content controls.
- `src/word/refreshCitations.ts` scans citations in document order, assigns numeric labels, and rebuilds bibliography output.
- `src/word/renderBibliography.ts` renders numeric bibliography entries.
- `src/ui/taskpane/*` contains the task pane UI.

## Start the Refx Local Bridge

The bridge is started automatically by the Tauri app after the local database is initialized.

Development endpoints:

- `GET http://127.0.0.1:38474/health`
- `GET http://127.0.0.1:38474/works?query=thesis`
- `GET http://127.0.0.1:38474/works/:workDocumentId/references?query=smith`
- `GET http://127.0.0.1:38474/references/doc%3A...`
- `GET http://127.0.0.1:38474/references/ref%3A...`

The bridge binds only to `127.0.0.1`. The Word add-in first lists Refx documents whose type is `my_work`; the user chooses the work/manuscript linked to the current Word document. After that, the add-in only sees references attached to that work via Refx `work_references`, preserving the user-defined Refx order.

Reference IDs are stable and namespaced as `ref:<referenceId>`.

Returned metadata includes:

- `id`
- `citationKey`
- `title`
- `authors`
- `year`
- `journal`
- `booktitle`
- `publisher`
- `volume`
- `issue`
- `pages`
- `doi`
- `url`
- `bibtex`

If the bridge is unavailable, the task pane shows a clear offline message and does not expose mock references in normal use. For UI-only development, append `?mock=1` to the task pane URL to use the small mock dataset.

## Run the Add-in UI

From this folder:

```bash
pnpm install
pnpm manifests
pnpm certs:install
pnpm dev
```

If Word shows `This content is blocked because it is not signed by a valid security certificate`, the local HTTPS certificate is not trusted by Office. Run:

```powershell
pnpm --dir word-addin certs:install
pnpm --dir word-addin certs:verify
```

Then fully close Word and start it again. The Vite dev server uses the same trusted localhost certificate generated by `office-addin-dev-certs`.

The development manifest is `word-addin/manifest.xml`. Sideload it in desktop Word using Microsoft's Office add-in sideloading workflow for your platform.

### Windows Desktop Sideload Helper

If Word's add-in manager keeps asking you to activate the trusted catalog even after Trust Center is enabled, use the helper script to bypass the broken UI path:

```powershell
pnpm --dir word-addin trust:windows
```

The script:

- Copies `manifest.xml` into `Documents\RefxWordAddinCatalog`.
- Tries to create a local Windows share named `RefxWordAddinCatalog`.
- Registers `\\localhost\RefxWordAddinCatalog` as a trusted Office add-in catalog under the current Windows user.

If PowerShell is not running as Administrator, Windows may refuse to create the SMB share. In that case the script prints manual sharing steps. After the share exists, close all Word windows, start `pnpm --dir word-addin dev`, reopen Word desktop, then use `Home > Add-ins > Advanced > Shared Folder` and choose Refx.

### If Word's Add-ins UI Crashes

If opening `Advanced` shows `This app could not be started`, the Word add-ins shell is crashing before Refx loads. Use Microsoft's debugging sideload flow instead:

```powershell
pnpm --dir word-addin dev
```

In a second terminal:

```powershell
pnpm --dir word-addin debug:word
```

This uses `office-addin-debugging` to register and launch the manifest directly in desktop Word, avoiding the broken add-ins manager dialog. Stop that sideload session with:

```powershell
pnpm --dir word-addin debug:word:stop
```

## Citation State in Word

The add-in uses three Word primitives:

- Inline citation content controls tagged `refx-citation`.
- One bibliography content control tagged `refx-bibliography`.
- One document-level custom XML part in namespace `https://refx.app/word-citations/v1`.

The custom XML stores:

- `sources`: stable Refx source ids mapped to metadata.
- `citationGroups`: each inserted citation occurrence, with stable group id, source ids, and the Word content-control id.
- `settings`: numeric style, linked Refx `workDocumentId`, linked work title, and bibliography order mode.

## Refresh and Bibliography Rebuilding

Refresh is idempotent:

1. Load citation state from custom XML.
2. Scan all `refx-citation` content controls in Word document order.
3. Match each control to its citation group.
4. Assign numbers by first appearance of each unique source id.
5. Update every visible citation label.
6. Drop deleted citation groups from state.
7. Rebuild the bibliography in either Word first-appearance order or Refx My Work reference order, depending on the selected option.
8. Save the repaired state back to custom XML.

Because numbers are derived from document order, inserting a new citation between existing citations is safe. Refresh recomputes all labels and the bibliography.

If `Refx My Work reference order` is selected, refresh still computes citation numbers by Word first appearance, but the bibliography entries are sorted according to the selected work's current Refx reference order. This means reordering references in Refx impacts the Word bibliography after refresh without breaking numeric labels.

Optional task pane settings:

- `Reference table order` chooses either text appearance order or Refx order.
- `In-text style` chooses Number, Author (Year), Author, year, or Author.
- `Container` chooses square brackets `[ ]`, parentheses `( )`, or no container.

## Stability Model

The beta add-in only mutates the Word document from explicit button actions:

- `Insert`
- `Refresh citations`
- `Rebuild table`
- `Repair`

React/task-pane state changes do not write to the document. A document mutation guard prevents concurrent insert, refresh, repair, and bibliography operations, so repeated clicks cannot start overlapping Word API batches.

The refresh engine also avoids unnecessary writes:

- Citation labels are only replaced when the visible label is different.
- The custom XML part is only rewritten when the serialized citation state changed.
- The bibliography is only rebuilt for `Rebuild table`, `Repair`, and refresh calls that explicitly request it.

## Repair Command

Use `Repair` when a document was edited heavily, a citation was deleted manually, or the visible labels no longer match the stored state.

Repair will:

- Scan all live `refx-citation` content controls in document order.
- Reload and validate the custom XML state.
- Drop stale citation groups whose content controls were deleted.
- Restore labels from stable source ids.
- Rebuild the reference table.
- Save the repaired XML state only if it changed.

Malformed custom XML is normalized instead of crashing the add-in. If the bridge is offline and a missing source must be fetched, the task pane reports the bridge problem rather than silently failing.

## Diagnostics

For development logging, append `?debug=1` to the task pane URL. Logs include:

- bridge health
- load works
- load references
- insert citation
- refresh citations
- save custom XML
- rebuild bibliography
- repair citation state

Mock data is disabled by default. Append `?mock=1` only for UI-only development when Refx desktop is not running. The task pane badge shows `Mock` in that mode so it is not confused with real Refx data.

## Manual Test Scenarios

1. Insert A, then B, then A again. Refresh. Expected labels: `[1]`, `[2]`, `[1]`.
2. Insert A, B, C. Insert D between A and B. Refresh. Expected labels in document order: A `[1]`, D `[2]`, B `[3]`, C `[4]`.
3. Delete a citation content control and refresh. Expected: numbering and bibliography close the gap.
4. Insert or load a grouped citation group, then refresh. Expected: the group renders as one citation label and numbering remains stable. The beta UI currently inserts one source at a time, but the refresh engine preserves grouped groups already present in state.
5. Save, close, reopen the Word document, then refresh. Expected: state reloads from custom XML and labels/bibliography rebuild correctly.
6. Stop Refx desktop and click search/sync. Expected: clean disconnected status, no freeze, no mock data unless `?mock=1` is set.
7. Click `Refresh citations` repeatedly. Expected: only one mutation runs at a time and the task pane reports the operation result.
8. Choose `Number` and `[ ]`, refresh. Expected: labels render like `[1]`.
9. Choose `Author (Year)` and no container, refresh. Expected: labels render like `Smith (2024)`.
10. Choose `Author, year` and `( )`, refresh. Expected: labels render like `(Smith, 2024)`.
11. Reorder references inside the linked Refx My Work, choose `Refx My Work reference order`, refresh in Word. Expected: bibliography follows the Refx order while visible citation numbers remain consistent with Word first appearance.
